Features

Built for developers who care about secrets

sigyn replaces scattered .env files with an encrypted local store, a desktop UI for managing environments, and a CLI that injects variables directly into child processes.

🔒

AES-256-GCM-SIV

Secret values are encrypted at rest with nonce-misuse-resistant authenticated encryption. The master key never leaves macOS Keychain.

🤚

Touch ID Authentication

Unlock with Touch ID or device password via macOS LocalAuthentication. No app-managed passwords to remember or rotate.

📁

No .env Files

Environment variables are injected directly into child processes via Rust's Command API. Nothing is written to disk in plaintext.

⌨️

CLI Companion

List projects, preview effective environments, and launch processes with injected secrets from the terminal.

🎯

Multi-Environment

Define local, dev, staging, and prod environments per project. Mix presets with per-entry overrides for flexibility.

💻

Local-First & Native

Built with Tauri 2 and Rust. Data stays on your machine. No accounts, no cloud, no telemetry.

Security

Defense in depth, anchored in macOS

sigyn delegates authentication and key storage to the operating system rather than managing its own password or vault.

🔑 Key Hierarchy

macOS Keychain └─ master key (256-bit random, generated once) └─ per-value encryption (AES-256-GCM-SIV) ├─ random 96-bit nonce per encrypt call └─ ciphertext + auth tag → base64 in SQLite

⏱️ Session Management

The master key is cached in memory only while unlocked. After 5 minutes of inactivity the session expires and all key bytes are zeroized before the memory is released.

CLI

Your secrets, one command away

The bundled CLI authenticates independently through macOS, reads the same encrypted store as the desktop app, and injects environment variables without shell evaluation.

List projects
$ sigyn list
Preview environment
$ sigyn preview --project "retail-service"
Run with injected env
$ sigyn run --project "retail-service" -- uv run python -m retail_service
Reset (development only)
$ sigyn reset-test-data --confirm "delete all data"
⚠️ The CLI refuses to inject dangerous env vars like PATH, LD_*, DYLD_*, NODE_OPTIONS, and other process-loading variables for safety.
Install

Up and running in minutes

sigyn is currently installed from source. The install script builds the Tauri app, copies it to /Applications, and symlinks the CLI onto your PATH.

1

Prerequisites

Make sure you have these installed:

  • macOS
  • Node.js + npm
  • Rust toolchain
  • Xcode / CLT
2

Clone & Install

Clone the repository and run the install script:

git clone https://github.com/connorguy/sigyn.git
cd sigyn
./install.sh
3

Launch

Open the desktop app from /Applications or Spotlight, then unlock with Touch ID.

sigyn --help
4

Quick Start

Create a project, add environments and entries, then preview or run with the CLI.

sigyn run --project "my-app" -- npm start
